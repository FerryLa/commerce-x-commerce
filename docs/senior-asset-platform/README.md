# 시니어·주니어 일자리 연결 플랫폼 — 설계 개요

> "시니어의 무형자산을 숫자로 만든다"  
> 노인·청소년 일자리 정책 프레임 | 세대간 지식 매칭 인프라

---

## 1. 문제 정의

기존 사회는 시니어의 자산을 **돈 · 부동산 · 연금** 위주로만 평가합니다.

하지만 실제로는:

| 측정 안 된 자산 | 예시 |
|---|---|
| 인생 경험 | 30년 자동차 정비 노하우 |
| 조정 능력 | 노사갈등 중재 50회 경험 |
| 직무 노하우 | 스마트팩토리 공정 설계 |
| 감정 안정성 | 위기 상담 경험 |
| 지역 네트워크 | 군산 제조업 인맥 |

→ 이것이 더 큰 자산입니다. 단지 **측정이 안 됐을 뿐**입니다.

---

## 2. 솔루션 구조 (3-Layer)

```
┌──────────────────────────────────────────────────────┐
│  Layer 1: Knowledge Graph (Neo4j)                    │
│  사람·경험·지역·관계를 노드/엣지로 저장              │
│  "누구의 인생 경험이 누구의 현재 문제를 해결하는가"  │
├──────────────────────────────────────────────────────┤
│  Layer 2: Reputation Engine (SourceCred 응용)        │
│  기여 이벤트 수집 → 점수 계산 → 배지/레벨 부여      │
│  "시니어의 존엄을 데이터 자산으로 복권"              │
├──────────────────────────────────────────────────────┤
│  Layer 3: Matching Algorithm                         │
│  지역·세대·기술 적합도·성격 궁합 다차원 계산         │
│  "아이디어를 실제 서비스로 만드는 엔진"              │
└──────────────────────────────────────────────────────┘
             ↕ REST API / GraphQL
┌──────────────────────────────────────────────────────┐
│  Frontend: Streamlit → (Next.js 확장 예정)           │
│  Knowledge Graph 시각화 · 매칭 대시보드              │
└──────────────────────────────────────────────────────┘
```

## 3. 핵심 개념 정의

### 시니어 (Senior)
- 50세 이상, 은퇴 전후
- 보유 자산: 직무 경험, 지역 네트워크, 감정 안정성
- 역할: 지식 공급자, 멘토, 문제 해결사

### 주니어 (Junior)
- 18~35세, 사회 진입기
- 필요 자원: 실무 경험, 인생 가이드, 위기 극복 지혜
- 역할: 지식 수혜자, 에너지 공급자 (디지털 역량)

## 4. B2G 정책 연계

| 정책 | 연결 포인트 |
|---|---|
| 노인 일자리 사업 | 시니어 기여 점수 → 급여 대체 or 보완 |
| 청소년 취업 지원 | 주니어-시니어 멘토링 → 실무 경험 인증 |
| 지역 소멸 대응 | 지역 단위 지식 네트워크 구축 |
| 사회적 기업 지원 | 플랫폼 자체를 협동조합 형태로 운영 |
