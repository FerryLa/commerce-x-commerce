# Matching Algorithm 설계

> "좋은 자산이 있어도 연결이 안 되면 무용지물입니다."

---

## 1. 매칭 알고리즘 개요

시니어와 주니어를 단순히 "같은 지역"으로 연결하지 않습니다.
**다차원 유사도 계산**으로 최적 연결을 찾습니다.

---

## 2. 매칭 차원 (6개)

| 차원 | 가중치 | 계산 방법 |
|---|---|---|
| 기술 적합도 | 35% | 시니어 보유 Skill ↔ 주니어 NEEDS Skill 교집합 |
| 지역 근접성 | 20% | 동일 지역 1.0 / 인근 지역 0.6 / 원거리 0.2 |
| 문제-해결 매칭 | 25% | 시니어 SOLVED ↔ 주니어 IN_CRISIS 교집합 |
| 시간대 가용성 | 10% | 상호 가능 시간대 중복 비율 |
| 세대 선호도 | 5% | 멘토링 경험/선호 나이차 범위 |
| Reputation 보정 | 5% | 시니어 점수 높을수록 가산점 |

---

## 3. 최종 매칭 점수 공식

```python
match_score = (
    0.35 * skill_fit
  + 0.20 * region_proximity
  + 0.25 * problem_solution_fit
  + 0.10 * availability_overlap
  + 0.05 * generation_preference
  + 0.05 * (reputation_score / 100)
)
```

- 범위: 0.0 ~ 1.0
- 0.7 이상: 추천 (초록)
- 0.5~0.7: 가능 (노랑)
- 0.5 미만: 제외

---

## 4. 계산 예시

**주니어A (군산, 자동차 브랜드 창업, 번아웃)** 에게 최적 시니어 추천:

| 시니어 | 기술 | 지역 | 문제해결 | 최종 점수 | 추천 |
|---|---|---|---|---|---|
| 김○○ (군산, 자동차 정비, 창업 불안 해결 경험) | 1.0 | 1.0 | 1.0 | **0.91** | ✅ 강추 |
| 최○○ (군산, 노사 조정, 번아웃 해결 경험) | 0.2 | 1.0 | 1.0 | **0.61** | ✅ 가능 |
| 이○○ (전주, 교육 설계) | 0.0 | 0.6 | 0.0 | **0.19** | ❌ 제외 |

---

## 5. 그래프 탐색 방식 (Neo4j)

단순 필터링이 아니라 **그래프 경로 탐색**으로 간접 연결도 활용:

```
주니어A → NEEDS → 자동차 정비 ← HAS_SKILL ← 시니어X
주니어A → IN_CRISIS → 번아웃 ← SOLVED ← 시니어Y
주니어A → LIVES_IN → 군산 ← LIVES_IN ← 시니어Z
```

→ 이 3개 경로에 동시에 등장하는 시니어가 최고 매칭

---

## 6. 매칭 프로세스

```
1. 주니어 프로필 입력
   ↓
2. Knowledge Graph에서 연관 노드 탐색 (Neo4j BFS/DFS)
   ↓
3. 다차원 점수 계산
   ↓
4. 상위 3명 추천 + 이유 설명 ("당신이 필요한 X를 Y년 경험한 분입니다")
   ↓
5. 양방향 수락 → 채팅방 개설
   ↓
6. 멘토링 완료 → 만족도 평가 → Reputation 갱신
```

---

## 7. 향후 개선

- **강화학습**: 매칭 성공/실패 피드백으로 가중치 자동 조정
- **협업 필터링**: "비슷한 주니어가 만족한 시니어" 패턴 활용
- **임베딩**: 프로필 텍스트를 벡터화해 의미 기반 유사도 계산
